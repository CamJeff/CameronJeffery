Exact Change 2

Problem Statement

You are given a price of an object, the number bills/coins you have, and their denominations (all in cents). Nothing will exceed 10,000 cents (i.e $100). You want to print out both the amount of money you spent on the object, and the number of coins/bills you used to pay.

Input

  • An integer indicating the number of test cases.
  • The price of the item.
  • And integer $n$ representing the number of coins/bills you have.
  • The next $n$ lines are the denominations of the coins/bills.

Output

The total amount paid and the number of coins/bills used.

Reference Code (Python)


def main():
    T = int(input())
    for _ in range(T):
        P = int(input())  # price           
        n = int(input())  # number of coins/bills                 
        coins = [int(input()) for _ in range(n)] # denominations

        MAX = P + 10000
        INF = 10**9

        dp = [INF] * (MAX + 1)      
        dp[0] = 0

        for v in coins:      
            for s in range(MAX - v, -1, -1): 
                if dp[s] + 1 < dp[s + v]:
                    dp[s + v] = dp[s] + 1

        for pay in range(P, MAX + 1):
            if dp[pay] < INF:
                print(pay, dp[pay])
                break

if __name__ == "__main__":
    main()

Solution Explanation

Store the number of test cases in variable T. For each test case, store the price and number of coins/bills in varaibles P and n, and the denominaions into an array called coins.

Create a dp array with lots of entries initially with arbitrarily large numbers. Each dp[s] will hold the minimum number of items needed to form exactly a summed price of s. Set dp[0]=0.

For each coin denomination $v$, iterate backwards, and for each sum s, if we can make s with dp[s] coins, then we can make s+v using one more coin. Update dp[s+v] if we find a better (smaller) number of coins.

After processing all coins, scan the price all the way to max. The first reachable sum where dp[pay] != INF, is the minimum total amount we can pay. So print pay and the number of coins we use, as in the dp array.
