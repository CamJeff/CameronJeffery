Spiderman's Workout

Problem Statement

Given a sequence of numbers which represent how far Spiderman can travel up or down, he wants to know for which number should he travel up ($\text{U}$) or down ($\text{D}$) such that he starts and ends on street level, and has the smallest maximum height reached. Of course, he can't travel below street level.

Input

  • An integer $N$ for number of test scenarios
  • $2N$ lines specifying the test scenarios. First line is integer $M$, which is the number of distances, and the second line contains $M$ positive numbers/distances.

Output

The sequence of $\text{U}$ or $\text{D}$ moves so that he starts and ends on street level. Print IMPOSSIBLE if no solution exists.

Reference Code (Python)


def solve_case(a):
    m = len(a)
    S = sum(a)                        
    dp = [None] * (S + 1)              
    dp[0] = 0

    prev_h = [[-1] * (S + 1) for _ in range(m + 1)]
    prev_c = [['']  * (S + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        d = a[i - 1]
        ndp = [None] * (S + 1)
        for h in range(S + 1):
            if dp[h] is None: 
                continue

            nh = h + d
            if nh <= S:
                best = max(dp[h], nh)
                if ndp[nh] is None or best < ndp[nh]:
                    ndp[nh] = best
                    prev_h[i][nh] = h
                    prev_c[i][nh] = 'U'

            nh = h - d
            if nh >= 0:
                best = dp[h]
                if ndp[nh] is None or best < ndp[nh]:
                    ndp[nh] = best
                    prev_h[i][nh] = h
                    prev_c[i][nh] = 'D'
        dp = ndp

    if dp[0] is None:
        return "IMPOSSIBLE"

    res = []
    h = 0
    for i in range(m, 0, -1):
        res.append(prev_c[i][h])
        h = prev_h[i][h]
    return "".join(reversed(res))


def main():
    t = int(input())
    for _ in range(t):
        m = int(input())
        a = list(map(int, input().split()))
        print(solve_case(a))

if __name__ == "__main__":
    main()

Solution Explanation

Our array dp[h] is the smallest peak heaight seen along any legal path that ends at current height $h$. For each transition d, from height h, we can go: up to nh = h + d with new peak max(dp[h], nh), or down to nh = h - d (if nh >= 0) with peak unchanged at dp[h].

Save list of distances as variable a, number of steps as variable m, and S is the upper bound on any reachable height. Initialize dp array, which again stores minimal peak height among all legal ways to reach height h after current number of steps. Set dp[o]=0.

prev_h[i][h] stores height we came from step i-1 to reach height h after step i. prev_c[i][h] stores the move used at step i to reach height h (whether up or down).

Iterating through steps 1 to $m$, let d be distance at step i, and ndp will be next layer of DP after considering each step. Then, loop through all reachable heights h.

Let's first try up: nh = h + d, with the new peak being the maximum of the prior peak, dp[h], and the new height nh. If this improves minimal peak at nh, record it and remember the predecessor and move.

Then we try down: nh = h - d, and we must in addition check that new height nh is not negative, but update if this yields a smaller peak for nh.

By dp = ndp, we move to the next step. If dp[0] is None, then we know after all steps, a height of $0$ isn't reachable, so there is no legal solution. If not, we reconstruct the moves backwards, starting at i=m and h=0, reading the moves to reach h, then jumping to the previous height.
